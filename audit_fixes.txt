═══ CRITICO 1 — build.js sai com exit 0 quando env vars faltam ═══
Em build.js troca process.exit(0) por process.exit(1) quando SUPABASE_URL ou SUPABASE_ANON nao estao definidas. Isso faz o deploy do Vercel falhar visivelmente em vez de silenciosamente nao gerar o config.js.
Excecao: se existir um config.js local (ambiente de desenvolvimento) pode continuar com exit 0.
Nova logica:
if (!url || !anon) {
  if (fs.existsSync(outPath)) {
    console.warn('[build] Usando config.js local existente.');
    process.exit(0);
  }
  console.error('[build] ERRO: SUPABASE_URL e SUPABASE_ANON sao obrigatorias no Vercel.');
  process.exit(1);
}
